Trade Journal — Setup Guide

5 steps, all on Cloudflare's free tier. No photos, no R2, no credit card required.

1
Put these files in a GitHub repo
↗ Create new GitHub repo

Upload everything with this exact structure:

your-repo/ ├── index.html ← the app ├── schema.sql ← run once in D1 (Step 2) └── functions/ └── api/ ├── auth.js ← password check ├── trades.js ← daily trade data (D1) ├── rules.js ← entry rules (D1) ├── overrides.js ← capital-risked overrides (D1) └── sync.js ← optional Capital.com import

schema.sql isn't deployed — it's SQL you paste once in Step 2.

2
Create a D1 database Free

D1 is Cloudflare's free SQL database — this is what makes your journal sync across every device.

↗ Open D1

1. Click Create database

2. Name it exactly trade-journal → Create

3. Open it → Console tab

4. Paste the entire contents of schema.sql and click Execute.

3
Create a Pages project & bind the database
↗ Open Pages

Click Create application → Pages → Connect to Git and pick the repo from Step 1. Build command: leave blank. Output directory: / (root). Click Save and Deploy.

Then go to the project → Settings → Functions → Bindings and add a D1 database binding:

Variable name
DB
D1 database
trade-journal
Important: the variable name must be exactly DB (case-sensitive) — the code references it by that name. There is no R2 binding in this version.
4
Set your admin password

Settings → Environment variables → Add variable:

Variable name
ADMIN_PASSWORD
Value
your-secret-password

Check Encrypt, and set it for both Production and Preview.

This is the password you type to unlock editing on any device. Anyone without it can only view — they cannot save or change anything.
5
Re-deploy

Bindings and env vars only take effect on a fresh deploy. Go to Deployments → latest → Retry deployment (or push any change to GitHub).

✅ You're live!

Open the site, click 🔒 Admin top-right, enter your password, then click any day to log a trade. To enter your capital manually, click the Override button next to "Capital Risked".

Optional — Capital.com auto-import. You enter trades by hand, so you can ignore the Sync button. If you ever want it to pull closed trades automatically, add these env vars: CAP_API_KEY, CAP_IDENTIFIER, CAP_PASSWORD (and optionally CAP_DEMO=1). Without them, the Sync button simply does nothing useful — everything else works fine.
